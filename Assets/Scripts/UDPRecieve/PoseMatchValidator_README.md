# PoseMatchValidator 使用ガイド

## 概要
`PoseMatchValidator`は、Pythonから送信されるポーズ判定データを受信し、ゲーム内で表示されているポーズ画像の番号と照合するためのスクリプトです。

## 主な機能
1. **UDP通信**: Pythonからポーズマッチング情報を受信
2. **リアルタイム表示**: 検出されたポーズ番号をGUIで表示
3. **ゲーム連携**: DanceGameManagerとPlayerManagerに自動接続
4. **デバッグモード**: 詳細な情報をオンスクリーンで確認可能

## セットアップ方法

### 1. スクリプトの配置
- GameObjectを作成し、`PoseMatchValidator`スクリプトをアタッチ
- 名前は「PoseMatchValidator」などに設定

### 2. インスペクターでの設定

#### UDP設定
- **Receive Port**: デフォルトは5006（Pythonスクリプトの送信先と合わせる）

#### ゲーム連携（自動検索されます）
- **Dance Game Manager**: 自動で検索されます（手動設定も可能）
- **Player Manager**: 自動で検索されます（手動設定も可能）

#### デバッグ設定
- **Enable Debug GUI**: ONにすると画面左上にデバッグ情報を表示
- **Override Player Input**: ONにするとPythonの判定でPlayerManager.posetrueを自動更新

## 使い方

### 基本的な動作
1. ゲームを起動すると自動的にUDP受信を開始
2. Pythonからポーズデータが送信されると、リアルタイムで検出
3. 画面左上のGUIに以下の情報が表示されます：
   - UDP受信状態
   - 最終受信時刻
   - 検出されたポーズ番号（例: #1, #5, #10）
   - マッチ/ミスマッチの回数
   - ゲームマネージャーの接続状態
   - PlayerManagerのposetrue状態

### キーボード操作
- **[V]キー**: 自動判定反映のON/OFF切り替え
  - ONの場合、Pythonの判定結果でPlayerManager.posetrueが自動更新される
  - OFFの場合、表示のみで判定には影響しない
- **[I]キー**: コンソールに詳細情報を出力

## ゲーム内での確認方法

### Pythonとの番号照合
1. ゲーム内でポーズ画像が表示される
2. DanceGameManagerの`poseTexture[]`配列のインデックス（0始まり）がポーズ番号
3. Pythonからは`pose_number`として1始まりの番号が送信される
4. 画面左上のGUIで「検出ポーズ: #1, #3」のように表示される

### デバッグフロー
```
1. ゲーム起動
   ↓
2. PoseMatchValidator開始（自動）
   ↓
3. UDP受信待機（Port 5006）
   ↓
4. Pythonからデータ受信
   ↓
5. 検出されたポーズ番号を表示
   ↓
6. マッチ回数をカウント
```

## 期待されるPythonからのデータ形式
```json
{
  "type": "pose_matches",
  "matched_poses": [
    {
      "pose_number": 1,
      "angle_error": 12.5
    },
    {
      "pose_number": 3,
      "angle_error": 8.2
    }
  ],
  "timestamp": 1234567890.123,
  "angles": {
    "left_elbow": 145.2,
    "right_elbow": 142.8
  }
}
```

## トラブルシューティング

### データが受信されない場合
1. Pythonスクリプトが実行されているか確認
2. ポート番号が一致しているか確認（デフォルト: 5006）
3. ファイアウォールでUDP通信がブロックされていないか確認
4. コンソールログで「UDP受信開始」メッセージを確認

### ゲームマネージャーが見つからない場合
- DanceGameManagerやPlayerManagerがシーンに存在するか確認
- インスペクターで手動でアサインも可能

### 自動判定が反映されない場合
- **[V]キー**を押して「自動判定反映: ON」になっているか確認
- PlayerManagerが正しく接続されているか確認

## 仕様の詳細

### ポーズ番号の対応
- **ゲーム内**: `poseTexture[]`配列のインデックス（0, 1, 2, ...）
- **Python側**: `pose_number`（1, 2, 3, ...）として送信
- **GUI表示**: `#1, #2, #3`として表示

### 統計情報
- **マッチ回数**: ポーズが新たに検出された回数
- **ミスマッチ回数**: ポーズが検出されなくなった回数
